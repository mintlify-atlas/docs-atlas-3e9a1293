title: "Install RemoteTerm from source and connect your radio"
description: "Clone the repository, install dependencies with uv and Node.js, set your transport environment variable, and open the browser interface."
---

The fastest way to run RemoteTerm is to clone the repository and build it locally. This method gives you the most reliable serial communication, direct access to logs, and the simplest upgrade path. If you prefer containers, see the [Docker install](/installation/docker) guide instead.

## Prerequisites

Before you begin, make sure you have the following installed:

- **Python 3.11 or later**
- **Node.js LTS or current** (versions 20, 22, 24, or 25) — only needed if you are building the frontend yourself
- **uv** — the Python package manager used by RemoteTerm

Install uv if you do not have it:

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

You also need a MeshCore-compatible radio connected via USB serial, TCP, or BLE.

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone RemoteTerm and enter the project directory:

    ```bash
    git clone https://github.com/jkingsman/Remote-Terminal-for-MeshCore.git
    cd Remote-Terminal-for-MeshCore
    ```
  </Step>

  <Step title="Install dependencies">
    Install Python dependencies with uv, then build the frontend:

    ```bash
    uv sync
    cd frontend && npm install && npm run build && cd ..
    ```

    <Tip>
      If you are on lightweight hardware or do not want to install Node.js, you can fetch a prebuilt frontend instead of building it yourself:

      ```bash
      python3 scripts/setup/fetch_prebuilt_frontend.py
      ```

      This downloads and unpacks a prebuilt frontend into `frontend/prebuilt`. You can then start the server normally without having Node.js installed.
    </Tip>
  </Step>

  <Step title="Find your serial port">
    If you are connecting via USB serial, identify the port name for your operating system:

    <CodeGroup>

    ```bash Linux
    ls /dev/ttyUSB* /dev/ttyACM*
    ```

    ```bash macOS
    ls /dev/cu.usbserial-* /dev/cu.usbmodem*
    ```

    ```powershell Windows (PowerShell)
    Get-CimInstance Win32_SerialPort | Select-Object DeviceID, Caption
    ```

    ```bash WSL2
    # Run in an elevated PowerShell window (not WSL):
    winget install usbipd
    # Restart your console, then list devices:
    usbipd list
    # Make the device shareable (replace 3-8 with your bus ID):
    usbipd bind --busid 3-8
    # Attach the device to WSL (repeat each time you plug in):
    usbipd attach --wsl --busid 3-8
    # The device will appear in WSL as /dev/ttyUSB0 or /dev/ttyACM0
    ```

    </CodeGroup>

    If you are connecting via TCP or BLE, skip this step.
  </Step>

  <Step title="Start the server">
    Set the environment variable for your transport type and launch the server.

    <CodeGroup>

    ```bash Serial (explicit port)
    MESHCORE_SERIAL_PORT=/dev/ttyUSB0 uv run uvicorn app.main:app --host 0.0.0.0 --port 8000
    ```

    ```bash Serial (auto-detect)
    uv run uvicorn app.main:app --host 0.0.0.0 --port 8000
    ```

    ```bash TCP
    MESHCORE_TCP_HOST=192.168.1.100 MESHCORE_TCP_PORT=5000 uv run uvicorn app.main:app --host 0.0.0.0 --port 8000
    ```

    ```bash BLE
    MESHCORE_BLE_ADDRESS=AA:BB:CC:DD:EE:FF MESHCORE_BLE_PIN=123456 uv run uvicorn app.main:app --host 0.0.0.0 --port 8000
    ```

    </CodeGroup>

    On Windows, set environment variables as a separate statement before the launch command:

    ```powershell
    $env:MESHCORE_SERIAL_PORT="COM8"
    uv run uvicorn app.main:app --host 0.0.0.0 --port 8000
    ```

    <Warning>
      **Windows + MQTT:** Python's default Windows event loop (ProactorEventLoop) is incompatible with the MQTT libraries used by RemoteTerm. If you configure any MQTT integration, add `--loop none` to your uvicorn command:

      ```powershell
      uv run uvicorn app.main:app --host 0.0.0.0 --port 8000 --loop none
      ```

      If you omit this flag, the app starts normally but MQTT connections will fail and the UI will display a toast with this same guidance.
    </Warning>
  </Step>

  <Step title="Open the browser interface">
    Navigate to [http://localhost:8000](http://localhost:8000) in your browser. The full messaging and management interface loads from there.

    A live API explorer is available at [http://localhost:8000/docs](http://localhost:8000/docs).
  </Step>
</Steps>

## Run as a systemd service (Linux)

On Linux, you can install RemoteTerm as a persistent service that starts on boot and restarts automatically on failure. Run the installer script from the repo root:

```bash
bash scripts/setup/install_service.sh
```

The script runs as your current user and installs from the directory where you cloned the repository. You can rerun it later to change transport, bot, or auth settings. If the service is already running, the installer stops it, rewrites the unit file, reloads systemd, and restarts it with the new configuration.

<Note>
  For the full systemd service workflow, post-install operations, HTTPS setup, and troubleshooting, see the advanced setup guide.
</Note>
